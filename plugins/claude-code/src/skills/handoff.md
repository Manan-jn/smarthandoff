name: handoff
description: Generate a Smart Handoff briefing to continue this session in another AI tool.
  Use when approaching rate limits, context window limits, or wanting to switch tools.
  Supports: gemini, codex, cursor, claude, chatgpt
user_invocable: true
args: "[--to <tool>] [--mode lean|rich] [--preview] [--summarize]"
---

Generate a Smart Handoff briefing for continuing this session in another AI tool.

## Step 1: Read the current conversation

**If `--summarize` is passed:** Do not use pattern matching. Read the full conversation with deep understanding. Write the handoff fields as you would write a PR description — from comprehension, not extraction rules.
- goal.title: what would you name a PR for this work? Under 80 chars.
- goal.description: 3–5 sentences covering what was built, the approach, and current status including the "why"
- decisions: real choices including implicit ones ("tried X, switched to Y because Z"). Include technical trade-offs. Remove meta-commentary. Max 8.
- blockers: the actual unresolved problem, not just the last message. Empty array if the session ended cleanly.
- filesChanged: for each file, write what changed and why — not just what the file is
- nextSteps: specific and ordered. Name the file, function, or command.

**Otherwise (default):** Scan this entire session for:
- The original user goal (first substantial message they sent)
- All files written or edited (look at Write/Edit/MultiEdit tool calls — get the file paths)
- The current blocker or unresolved issue (last user message + any error in last assistant message)
- Key decisions made (look for "decided", "choosing", "not using", "rejected", "rather than", "instead of")
- The specific next action needed (last assistant message often states this)

## Step 2: Extract project context

- Read CLAUDE.md if present (use the Read tool)
- Read package.json for stack info (name, dependencies)
- Note the test command if mentioned anywhere in the session

## Step 3: Determine target tool

Parse --to from the arguments. Default: gemini.

Target options:
- **gemini**: generous format (50K tokens), include full CLAUDE.md, add note "Search for current docs on: [npm packages mentioned]"
- **codex**: tight format (under 8K tokens), exact file:line for errors, test command prominent, TASK:/FILE:/FAILING:/VERIFY: prefixes
- **cursor**: @filename references for changed files, MDC format note, clean markdown, omit pipe commands
- **claude**: shortest format (omit CLAUDE.md — already on disk), include memory fragments if any
- **chatgpt**: two labeled sections: "SYSTEM PROMPT:" and "FIRST MESSAGE:", human-readable

## Step 4: Generate the briefing

Format the briefing based on the target tool. Keep it under 15K tokens total.

### For gemini (default):

```
## CONTEXT (resuming from claude-code)
*Generated by /handoff — [timestamp]*

### Goal
**[first user message title]**
[first user message, up to 500 chars]

### Files changed
**[file path]** (modified, high)
[what was done in this file]

### Open blocker
[last user message or error, if any]
Error: `[exact error text if present]`
Location: [file:line if present]

### Decisions made (do not re-suggest)
- [decision 1]
- [decision 2]

### Next step
[specific next action]

### Stack
[stack from package.json]
Test: `[test command if known]`

### Project instructions (CLAUDE.md)
[CLAUDE.md content if within budget]

*Note: Search for current docs on: [npm packages]*
```

### For codex:
```
TASK: [goal title]
FILE: [path] (modified)
FAILING: [file:line]
  Error: [error message]
HYPOTHESIS: [suggested next step]
VERIFY: [test command]
DECISIONS: [decision 1]; [decision 2]
STACK: [stack]
SCOPE: Do not change files outside [dirs]
```

### For cursor:
Mention that the user should create `.cursor/rules/handoff.mdc` with the content, or run `smarthandoff route --to cursor` to do it automatically.

### For claude:
Short format — task status, files done, blocker, decisions, next step. No CLAUDE.md.

### For chatgpt:
```
SYSTEM PROMPT:
You are a senior software engineer continuing work from an AI coding session. Stack: [stack]. Follow existing patterns. Do not refactor what is already working.

---

FIRST MESSAGE:
I'm continuing a coding session. Rate limit hit on Claude, switching to you.

**What we were building:** [goal]
...
```

## Step 5: Output

1. Print the briefing in a markdown code block
2. Write the briefing to `.smarthandoff/latest.md` using the Write tool
3. Print:
```
---
Saved to: .smarthandoff/latest.md
Run: [the exact command to launch the target tool with this briefing]
```

The run commands are:
- gemini: `cat .smarthandoff/latest.md | gemini -i "You are resuming a coding task. Context is above."`
- codex: `cat .smarthandoff/latest.md | codex exec "Fix the failing issue. Context above." --sandbox workspace-write`
- cursor: `smarthandoff route --to cursor` (writes the rules file)
- claude: `claude` (paste the briefing as first message)
- chatgpt: Open ChatGPT, paste system prompt, then paste first message
```
